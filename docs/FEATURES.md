# Features — Enough

Status legend: ✅ done · 🔨 in progress · 📋 planned · 🧊 backlog (post-v1.0)

## Core loop
- 📋 Day/shift screen with incoming card queue
- 📋 Request card with 2–3 responses + defer/dismiss
- 📋 Visible resources: Time, Energy, Patience (HUD)
- 📋 Response costs spend resources
- 📋 Hidden states (Connection, Security, Partnership, Wellbeing, Standing)
- 📋 Day boundary: end-of-day rollover, partial overnight refill
- 📋 Deferred/unaddressed needs resurface (followups)

## Burnout / fatigue
- 📋 Fatigue level derived from Energy+Patience
- 📋 Warm options gate/lock when reserves low
- 📋 Harsher options appear when depleted
- 📋 UI degradation (desaturation, slowed animation, terser text)
- 📋 Audio dulling (low-pass / tempo) when tired

## Narrative
- 📋 Interstitial quiet beats between days (state-selected)
- 📋 Endings (3–5 variants) computed from accumulated tone
- 📋 Day-open framing lines

## Content pipeline
- 📋 JSON event/card schema + authoring guide
- 📋 EventDeck loader + day/condition filtering
- 📋 Content validation (headless)
- 📋 Full week of authored content (~40–60 cards)

## Shell / meta
- 📋 Main menu (new / continue / settings)
- 📋 Settings (audio, haptics, text size, reduce-motion, reset)
- 📋 Save / continue (autosave at day boundary)
- 📋 Pause

## Polish

- 📋 Type-on dialogue with pacing
- 📋 Sound design (ambient, motif, diegetic interrupts)
